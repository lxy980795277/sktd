## 图片目录扩展名修复（/Users/liuxinyu/Desktop/images）
- 问题：用户图片在 Mac 能打开，Windows 部分打不开。根因是扩展名与真实格式不符：
  - `home/featured-banner/*.jpg` 实为 WebP（Windows 10+ 能开，所以没报错）
  - `home/hero-carousel`、`home/about-section`、`home/advantages-section`、`products/*/detail.jpg` 实为 AVIF，Windows 照片查看器按 .jpg 解码失败 → 打不开
  - `home/market-film-section/*.jpg` 是真 JPEG，不受影响
- 处理：按真实格式批量 mv 扩展名
  - 43 个 AVIF → `.avif`
  - 4 个 WebP → `.webp`
  - 共 47 个文件改名完成，验证无 mismatch
- 注意：若后续接入前端项目（home/products 结构像页面素材），代码中对这些图片的引用路径需同步更新后缀

## 方案 B 二次处理：全部转真 JPEG
- 用户确认后，将 47 个 .avif/.webp 全部用 macOS `sips` 转成 JPEG（quality=90）
  - 命令：`sips -s format jpeg -s formatOptions 90 <src> --out <dst.jpg>`
  - 转换成功后删除原文件
- 结果：images/ 下所有图片统一为真 JPEG（共 125 个 .jpg），Windows 可直接双击打开，无需装解码器
- 体积：AVIF → JPEG 约增加 30%（如 241K → 316K），可接受
- 经验：macOS 自带 sips 支持读 avif/webp，不依赖 ffmpeg/imagemagick
